# v0.2.0 — On-Device Transcription

**Target version:** 0.2.0
**Builds on:** v0.1.0

## Overview

Adds on-device transcription using whisper.cpp. After a recording session ends and all chunks are encoded, the session is queued for transcription. When the device is idle, it works through the queue — concatenating the session's Opus files and transcribing to a `transcript.md` compatible with earshot-tui. Transcription is enabled by default and runs silently in the background without competing with recording.

See [ADR-0010](../docs/adr/0010-on-device-transcription.md) and [transcription requirements](../docs/requirements/transcription.md) for full design rationale.

---

## Scope

### New Functionality

- **Transcription queue** — implicit FIFO queue derived from filesystem state; sessions with `.opus` files and no `transcript.md` are pending
- **Transcription process** — all session `.opus` chunks concatenated via `ffmpeg` and piped to `whisper-cli`; no intermediate file written
- **Transcript output** — `transcript.md` per session in earshot-tui format with timestamped segments
- **Transcribing device state** — LED amber slow pulsate while transcribing; returns to solid green when queue empties
- **Whisplay LCD: Transcribing screen** — shows queue position and current session timestamp; brief completion screen on queue empty
- **Recording interrupts transcription** — button press cancels transcription immediately; session re-queued to front
- **Transcription failure handling** — failed sessions remain at front of queue; retried on next idle window; logged to journal
- **Installer: whisper.cpp setup** — installs whisper.cpp binary and downloads `ggml-tiny.en-q5_1.bin` by default; `--no-transcription` flag to skip

### Configuration

New `[transcription]` section in `config.toml`:

```toml
[transcription]
enabled = true
model = "tiny.en"
threads = 2
```

### Out of Scope (not in this release)

- Diarization
- `require_transcript_before_offload` config option (backlog)
- Model selection beyond `tiny.en` and `base.en`
- Real-time / live transcription during recording

## Implementation Tasks

### 1. Installer (`installer/install.sh`)

- [ ] Add whisper.cpp installation step (pre-built aarch64 binary; cmake build from source as fallback)
- [ ] Download `ggml-tiny.en-q5_1.bin` to `~/.local/share/earshot/models/`
- [ ] Add `--no-transcription` flag to skip whisper.cpp and model download
- [ ] Write `[transcription]` defaults to `config.toml`

### 2. Config (`earshot/config.py`)

- [ ] Add `TranscriptionConfig` dataclass (`enabled`, `model`, `threads`)
- [ ] Parse `[transcription]` section with defaults
- [ ] Validate `model` is a known value; validate `threads` is a positive integer

### 3. Transcription module (`earshot/transcription/`)

- [ ] `queue.py` — scan recordings directory; return pending sessions (FIFO by directory name)
- [ ] `process.py` — build ffmpeg concat command; invoke `whisper-cli` subprocess; capture JSON output
- [ ] `writer.py` — convert whisper.cpp segment output to earshot-tui `transcript.md` format
- [ ] Failure handling — on non-zero exit or parse error, leave session pending; log to journal

### 4. State machine (`earshot/app.py`)

- [ ] Add `TRANSCRIBING` state
- [ ] On entering idle: check transcription queue; transition to `TRANSCRIBING` if non-empty
- [ ] On button press during `TRANSCRIBING`: cancel subprocess, re-queue session to front, transition to recording
- [ ] On transcription complete: re-check queue; transition to idle if empty, or start next session
- [ ] On boot: existing queue scan covers pending sessions automatically

### 5. HAL — LED (`earshot/hal/`)

- [ ] Add amber colour (`#FFB300`) to colour constants
- [ ] Add slow pulsate variant with ~1.5–2s cycle (distinct from existing ~1s pulsate) for transcription state
- [ ] Wire to `TRANSCRIBING` state in all HAL implementations (ReSpeaker, Whisplay, Stub)

### 6. HAL — Display (`earshot/hal/whisplay.py`)

- [ ] Implement `TRANSCRIBING` screen: Zone A `TRANSCRIBING` amber; Zone C queue position; Zone D session timestamp
- [ ] Implement queue-complete screen: Zone A `DONE` amber; Zone C `Transcription complete`; Zone D session count; auto-dismiss after 3s

### 7. Tests

- [ ] `tests/test_transcription_queue.py` — queue detection from fixture session directories
- [ ] `tests/test_transcription_writer.py` — segment JSON → `transcript.md` format correctness
- [ ] `tests/test_transcription_process.py` — subprocess invocation (mocked whisper-cli)
- [ ] Extend `tests/test_usb_offload.py` — verify sessions with and without `transcript.md` offload correctly

## Acceptance Criteria

- [ ] A completed session with `transcription.enabled = true` produces a `transcript.md` in the session directory when the device returns to idle
- [ ] `transcript.md` format matches earshot-tui output (header, duration, timestamped segments)
- [ ] Pressing the record button while the amber LED is pulsating cancels transcription and begins recording without delay
- [ ] After recording, the cancelled session is the first to be transcribed when the device next returns to idle
- [ ] USB offload moves sessions regardless of whether `transcript.md` is present
- [ ] Setting `transcription.enabled = false` results in no amber LED state and no `transcript.md` files
- [ ] `--no-transcription` installer flag skips whisper.cpp and model download entirely
- [ ] All new tests pass locally (stub HAL)
- [ ] Manually verified on Pi 4B: 15-minute session transcribed to `transcript.md` within expected time window
- [ ] Manually verified on Pi Zero 2W: `tiny.en` model fits in available RAM; transcription completes
